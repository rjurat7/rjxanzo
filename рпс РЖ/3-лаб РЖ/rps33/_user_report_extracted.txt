1. Цель работы
Разработка для программы графического пользовательского интерфейса и интеграция базы данных для веб-приложений.
2. Описание задания по варианту
В репозитории проекта, в выбранной системе контроля версий, создать новую ветку (develop-lab3).
Развернуть базу данных (любую на выбор, SQL или NoSQL СУБД). Создать структуру базы данных, реализующую хранение массивов целых чисел (как отсортированных, так и не отсортированных).
Реализовать интеграционные тесты по работе с базой данных:
Тест добавления 100/1000/10000 массивов в базу данных программы (размер и наполнение массивов генерируется случайным образом), который в качестве ответа выводит флаг успешного / не успешного выполнения и время работы;
Тест выгрузки и сортировки 100 случайных массивов из базы данных, который в качестве ответа выводит флаг успешного / не успешного выполнения, общее время работы теста и среднее время работы с 1 массивом (данный тест должен быть запущен 3 раза для базы на 100, 1000 и 10000 записей);
Тест очистки базы данных, который в качестве ответа выводит флаг успешного / не успешного выполнения и общее время работы теста (данный тест должен быть запущен 3 раза для базы на 100, 1000 и 10000 записей).
Тесты должны обращаться к программным функциям создаваемой программы, но не должны быть частью самой программы. Т.е. это может быть отдельный проект с тестами, которые запускаются отдельно, например, через консоль. Для тестов должна быть отдельная база данных.
Реализовать графический пользовательский интерфейс (стек технологий любой на выбор). Получившийся интерфейс должен хотя бы отчасти соответствовать следующий эвристикам ui/ux:
видимость статуса системы;
соответствие между системой и реальным миром; управляемость; согласованность;
гибкость; минимализм в дизайне;
помощь пользователю в выявлении ошибок;
наличие справки.
В итоге необходимо реализовать desktop-приложение, клиент-серверное приложение или веб-приложение, учитывающее основные требования к функционалу, описанные ниже.
3. Описание используемых технологий
В ходе работы была реализована база данных SQLite - встраиваемая SQL СУБД, которая хранит всю базу данных в одном файле.
Достоинства:
Встраиваемость - не требует отдельного сервера, работает как библиотека;
Простота развертывания - один файл sorting.db;
Полноценная SQL-поддержка - индексы, транзакции, внешние ключи;
Кросс-платформенность - работает на Windows, Linux, macOS;
Производительность - быстрая работа с локальными данными;
Нулевая конфигурация - не требует администрирования;
Портативность - файл БД можно копировать между системами;
Бесплатность - открытый исходный код.
Недостатки:
Ограниченная многопользовательская поддержка - лучше всего подходит для однопользовательских приложений;
Максимальный размер БД - теоретически до 140 ТБ, практически ограничения файловой системы;
Ограниченные типы данных - по сравнению с серверными СУБД.
