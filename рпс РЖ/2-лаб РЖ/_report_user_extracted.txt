4. Расчёт лучшей и худшей сложности алгоритма
Описание алгоритма:
Шейкерная сортировка - улучшенная версия пузырьковой сортировки, которая проходит массив в обоих направлениях. На каждой итерации максимальные элементы "всплывают" вправо, а минимальные - влево.
Лучший случай: O(n)
Условие: Массив уже отсортирован в правильном порядке
Расчёт:
Первая итерация внешнего цикла:
Проход слева направо: выполняется (n-1) сравнений
Ни одного обмена не происходит (массив уже отсортирован)
Проверка условия !swapped = true → алгоритм завершает работу
Общее количество операций:
Сравнения: (n-1)
Обмены: 0
Внешние итерации: 1
T(n) = (n-1) сравнений + 0 обменов + константные операции
T(n) = n - 1 + C
O(T(n)) = O(n)
Худший случай: O(n²)
Условие: Массив отсортирован в обратном порядке
Расчёт:
Количество внешних итераций: ⌈n/2⌉ (на каждой итерации границы сужаются с двух сторон)
Анализ по итерациям:
Итерация 1:
Проход слева направо: (n-1) сравнений, (n-1) обменов
Проход справа налево: (n-2) сравнений, (n-2) обменов
Итого: (2n-3) сравнений, (2n-3) обменов
Итерация 2:
Проход слева направо: (n-3) сравнений, (n-3) обменов
Проход справа налево: (n-4) сравнений, (n-4) обменов
Итого: (2n-7) сравнений, (2n-7) обменов
Общее количество операций:
T(n) ≈ n + (n-2) + (n-4) + ... + 2 (для чётного n)
Асимптотическая сложность:
O(T(n)) = O(n(n + 2)/4) = O(n²)
Средний случай: O(n²)
Обоснование: В среднем массив частично отсортирован, но количество инверсий всё равно пропорционально n², что приводит к квадратичной сложности.

Шейкерная сортировка демонстрирует хорошую производительность на небольших и частично отсортированных массивах, но для больших объёмов данных рекомендуется использовать более эффективные алгоритмы с лучшей асимптотической сложностью.
